# Security Policy

## Supported versions

Only the latest `main` branch is supported during the pre-1.0 phase.

## Reporting a vulnerability

Open a private security advisory if the repository is public. If private
advisories are unavailable, contact the maintainer through the repository issue
tracker without posting secrets or private lecture data.

## Data handling

Do not attach real classroom materials, private transcripts, patient data,
student data, OAuth files, API keys, provider caches, or copyrighted decks to
public issues. Use synthetic fixtures instead.
